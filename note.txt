HOW TO DEPLOY CLIENT SIDE ON VERCEL
=====================================

Prerequisites:
- Vercel account (sign up at vercel.com)
- Git repository (GitHub, GitLab, or Bitbucket)

Method 1: Deploy via Vercel Dashboard
--------------------------------------
1. Push your code to a Git repository (GitHub recommended)
2. Go to https://vercel.com and sign in
3. Click "Add New" → "Project"
4. Import your Git repository
5. Configure project settings:
   - Framework Preset: Vite (or React if detected)
   - Root Directory: client
   - Build Command: npm run build
   - Output Directory: dist
6. Add environment variables (if any) from client/.env.example
7. Click "Deploy"

Method 2: Deploy via Vercel CLI
--------------------------------
1. Install Vercel CLI:
   npm install -g vercel

2. Navigate to client directory:
   cd client

3. Login to Vercel:
   vercel login

4. Deploy:
   vercel

5. For production deployment:
   vercel --prod

Important Configuration:
------------------------
- Ensure client/package.json has correct build scripts
- Add vercel.json in client folder if custom configuration needed:
  {
    "buildCommand": "npm run build",
    "outputDirectory": "dist",
    "devCommand": "npm run dev",
    "installCommand": "npm install"
  }

- Update API endpoints in client code to point to deployed backend server
- Set environment variables in Vercel dashboard under Project Settings → Environment Variables

Post-Deployment:
----------------
- Vercel will provide a live URL (e.g., your-app.vercel.app)
- Configure custom domain if needed in Project Settings → Domains
- Enable automatic deployments for Git pushes
